



# 简易版待办事项系统需求文档（PRD）
## 一、项目概述
### 1.1 产品定位
基于用户指定的“三侧边栏+自定义清单”核心结构，打造一款**层级清晰、操作直观**的Web端待办事项管理系统。聚焦个人用户的任务分类管理需求，通过“动态记录、全部任务汇总、多级任务清单”三大核心模块，实现任务的精细化管理与高效追踪。

### 1.2 目标用户
个人用户：需要清晰分类管理日常工作/生活任务、追踪任务动态、灵活创建多级任务清单的用户。

### 1.3 核心价值
- 结构清晰：三侧边栏布局，任务、清单、动态分离，信息层级一目了然；
- 灵活定制：支持自定义任务清单，满足不同场景的任务分类需求；
- 多级嵌套：任务支持无限层级子任务（建议限制最大5级，避免过深），适配复杂任务拆解场景；
- 动态追踪：自动记录所有任务操作，便于回溯任务变更历史。

## 二、功能需求
### 2.1 整体布局与导航
- **页面结构**：左侧固定三侧边栏（动态、全部任务、任务清单）+ 底部“新建分组”按钮 + 右侧主内容区（根据侧边栏选择展示对应模块）；
- **导航规则**：点击侧边栏项切换主内容区，当前激活项高亮显示；底部“新建分组”按钮固定在侧边栏底部，点击弹出“新建任务清单”弹窗。

### 2.2 动态模块
- **核心功能**：自动记录系统内所有任务/清单的CRUD操作，生成动态流；
- **动态内容**：每条动态包含“操作类型（创建/编辑/删除/状态变更）、操作对象（任务/清单名称）、操作人（当前登录用户）、操作时间、变更详情（如“任务状态从‘待处理’改为‘已完成’”）”；
- **展示规则**：按操作时间倒序排列（最新动态在前），支持滚动加载历史动态；
- **筛选功能**：支持按“操作类型（创建/编辑/删除）、操作对象类型（任务/清单）”筛选动态。

### 2.3 全部任务模块
- **核心定义**：包含所有任务的汇总视图，即“任务清单内的任务”+“直接在全部任务中创建的独立任务”的总和；
- **任务创建**：支持直接在本模块创建独立任务（不属于任何清单），创建入口为右上角“新建任务”按钮；
- **任务管理**：
  - 支持编辑任务信息（标题、描述、截止时间、优先级等）、删除任务、标记任务状态（待处理→进行中→已完成）；
  - 支持将独立任务“移动到”指定任务清单（通过任务操作菜单选择）；
  - 支持查看任务所属清单（若有），点击清单名称可跳转至对应清单；
- **视图与筛选**：
  - 支持2种视图：列表视图（默认，按创建时间排序）、看板视图（按任务状态分组展示）；
  - 筛选功能：支持按“状态、优先级、截止时间、所属清单（或独立任务）”筛选；
  - 排序功能：支持按“创建时间、截止时间、优先级”升序/降序排序。

### 2.4 任务清单模块
#### 2.4.1 清单管理
- **清单创建**：通过底部“新建分组”按钮创建自定义清单，需输入清单名称（限20字内），创建后显示在“任务清单”侧边栏；
- **清单操作**：支持清单重命名、删除（删除前需确认，删除后清单内所有任务自动转为独立任务，移至“全部任务”）；
- **清单排序**：支持拖拽调整侧边栏中清单的显示顺序（默认按创建时间排序）。

#### 2.4.2 清单内任务管理
- **任务创建**：进入具体清单后，通过右上角“新建任务”按钮创建清单内任务；
- **任务基本信息**：
  - 必填项：任务标题（限50字内）、任务状态（默认“待处理”）；
  - 选填项：截止时间（支持日期+时间选择）、优先级（高/中/低，默认“中”）、详细描述（限500字内）；
- **子任务管理**：
  - 支持为任务添加子任务，子任务支持再次添加子任务（建议限制最大嵌套层级为5级）；
  - 子任务继承父任务的清单归属，不可独立移动到其他清单；
  - 支持单独标记子任务状态（待处理/进行中/已完成），父任务状态可手动或自动关联（如“所有子任务完成后，父任务自动标记为已完成”，可在设置中开关此规则）；
  - 支持拖拽调整同层级任务/子任务的顺序；
- **任务操作**：支持编辑任务信息、删除任务、标记状态变更、添加/编辑/删除子任务。

### 2.5 任务与子任务通用功能
- **附件支持**：所有任务/子任务支持上传10MB以内的文档、图片附件（最多3个）；
- **备注功能**：支持为任务/子任务添加文字备注（限200字内）；
- **状态流转**：任务/子任务状态支持“待处理→进行中→已完成”三级流转，不支持反向流转（如需反向，需手动修改）；
- **截止时间提醒**：任务/子任务截止前1小时发送系统通知（可在设置中关闭）。

### 2.6 数据管理
- **任务归档**：已完成的任务支持手动归档，归档后从列表中隐藏，可在“归档中心”查看或恢复；
- **数据导出**：支持将单个清单的任务或全部任务导出为Excel文件（包含任务标题、状态、优先级、截止时间、所属清单、子任务层级等字段）；
- **本地存储**：任务数据自动本地备份，支持浏览器刷新后数据恢复。

## 三、非功能需求
### 3.1 性能要求
- 页面加载时间≤1秒，任务/子任务操作（创建/编辑/状态变更）响应时间≤500ms；
- 支持单个用户创建最多100个任务清单，单个清单最多容纳500条任务（含子任务）。

### 3.2 兼容性要求
- 支持主流浏览器（Chrome、Edge、Firefox 最新3个版本）；
- 支持移动端浏览器自适应访问（核心功能可正常操作）。

### 3.3 易用性要求
- 操作流程极简：创建任务/清单步骤≤2步，添加子任务步骤≤1步；
- 界面设计：采用简约风格，色彩以白/浅灰为主，优先级用红（高）、黄（中）、绿（低）区分，子任务通过缩进层级清晰展示；
- 交互反馈：所有操作（如创建成功、删除确认）均有明确的视觉/文字反馈。

## 四、界面原型建议（核心页面）
### 4.1 首页（默认全部任务视图）
- **左侧边栏**：
  - 顶部：“动态”“全部任务”“任务清单”三个导航项，当前激活“全部任务”；
  - 中部：“任务清单”列表（包含用户创建的所有清单，如“工作任务”“个人事务”）；
  - 底部：固定“新建分组”按钮（+ 图标，悬停显示“新建任务清单”）；
- **右侧主内容区**：
  - 顶部：筛选栏（状态、优先级、截止时间、所属清单下拉选择器）+ 排序按钮 + “新建任务”按钮；
  - 中部：任务列表（或看板），独立任务和清单内任务混合展示，清单内任务左侧显示所属清单标签；
  - 底部：分页或滚动加载。

### 4.2 任务清单详情页
- **左侧边栏**：当前清单高亮显示；
- **右侧主内容区**：
  - 顶部：清单名称 + 任务统计（如“10个任务，3个已完成”）+ “新建任务”按钮；
  - 中部：任务列表，支持折叠/展开子任务，子任务通过缩进层级展示；
  - 任务项操作：悬停显示“编辑、删除、添加子任务”按钮。

### 4.3 动态页
- **右侧主内容区**：
  - 顶部：筛选栏（操作类型、对象类型）；
  - 中部：动态流列表，每条动态包含操作图标、操作描述、操作时间，如“[10:30] 创建了任务‘完成需求文档’”。

### 4.4 新建任务/清单弹窗
- **新建任务弹窗**：
  - 标题输入框 + 截止时间选择器 + 优先级选择器 + 描述输入框 + “添加子任务”按钮（可点击展开子任务输入框）+ “确认创建”按钮；
- **新建清单弹窗**：
  - 清单名称输入框 + “确认创建”按钮。

## 五、功能优先级说明
| 功能模块     | 优先级 | 说明                               |
| ------------ | ------ | ---------------------------------- |
| 三侧边栏布局 | 高     | 核心结构，必须优先实现             |
| 任务清单管理 | 高     | 核心功能，支持自定义清单创建与管理 |
| 任务与子任务 | 高     | 支持多级子任务，核心业务逻辑       |
| 全部任务汇总 | 中     | 汇总展示所有任务，提升查询效率     |
| 动态记录     | 中     | 辅助功能，便于追踪任务变更         |
| 筛选与排序   | 中     | 提升任务查找效率                   |
| 数据导出     | 低     | 辅助功能，按需实现                 |
